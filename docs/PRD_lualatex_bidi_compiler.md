# Mechanism PRD: LuaLaTeX Compilation & BiDi Engine

## 1. Description & Background
Compiling LaTeX programmatically is challenging, but compiling LaTeX that dynamically switches between Left-to-Right (English) and Right-to-Left (Hebrew) text is exceptionally complex. This mechanism is responsible for taking the sanitized Markdown/LaTeX strings, injecting them into a master `.tex` template alongside conditional cover sheet metadata, and orchestrating the local LuaLaTeX binaries to produce a flawless 15-page PDF document.

## 2. Expected Inputs and Outputs
* **Input:** 1. Processed `.tex` body string (generated by agents).
  2. A metadata dictionary (`topic`, `language`, `author`, `course`, `date`, `lecturer`).
  3. Local file paths to assets (e.g., `assets/graph.png`).
* **Output:** A compiled PDF file saved as `xxxxxxxx-eyyy.pdf` in the root directory.

## 3. Constraints, Alternatives, and Rationale (ADR)
* **Constraint:** Must support both purely English output and BiDi (Hebrew/English) output dynamically based on the CLI prompt.
* **Alternative 1: XeLaTeX.** A powerful unicode engine. *Rejected* because its BiDi package support (specifically `bidi` vs `polyglossia`) can be brittle when mixed with complex TikZ diagrams and certain verbatim code blocks.
* **Alternative 2: pdfLaTeX with `babel`.** The traditional approach. *Rejected* due to extremely poor native support for UTF-8 Hebrew font shaping.
* **Selected Architecture: LuaLaTeX.** Chosen for its modern, robust font shaping engine and superior memory limits. It allows seamless integration of the `babel` package with `bidi=basic` or `luabidi`, ensuring Hebrew text aligns RTL while mathematical formulas (`amsmath`) remain perfectly LTR.

## 4. Test Scenarios & Acceptance Criteria
* **Scenario A (Conditional Cover Sheet):** Pass the metadata dictionary with `course` and `date` set to `None`.
  * *Success:* The Jinja2/string templater completely omits these lines from the `.tex` cover sheet. The PDF compiles without empty gaps or placeholder syntax (`{{course}}`).
* **Scenario B (Multi-Pass Compilation):** Inject a Table of Contents (`\tableofcontents`) and a `.bib` file containing 3 citations.
  * *Success:* The Python script executes the compile loop exactly 4 times (`lualatex` -> `biber` -> `lualatex` -> `lualatex`). The final PDF contains a populated TOC and clickable bibliography links.
* **Scenario C (Language Fallback):** Run the compilation with the `language` flag set to `english`.
  * *Success:* The script omits the Hebrew BiDi packages from the preamble, resulting in a slightly faster compile time and a pure LTR document.
