# Mechanism PRD: Asset & Structure Injector (Markdown to LaTeX)

## 1. Description & Background
CrewAI agents naturally output responses in Markdown format. However, standard Markdown cannot support the complex typesetting required by the assignment (e.g., TikZ graphs, specific `\begin{equation}` blocks, and strict table dimensions). This mechanism acts as the bridge: it parses the raw Markdown from the Editor agent, escapes dangerous LaTeX characters, and structurally converts markdown elements (tables, bolding, images) into native LaTeX syntax.

## 2. Expected Inputs and Outputs
* **Input:** A raw Markdown string generated by the CrewAI pipeline, referencing local assets (e.g., `![Graph](assets/graph.png)`).
* **Output:** A sanitized, fully compliant `.tex` string ready to be injected into the `\begin{document}` block of the master template.

## 3. Constraints, Alternatives, and Rationale (ADR)
* **Constraint:** Must securely escape LaTeX reserved characters (`&`, `%`, `$`, `#`, `_`, `{`, `}`) used in regular text, without breaking actual intended math formulas.
* **Alternative 1: `pypandoc` Wrapper.** Use the Pandoc library to convert Markdown to LaTeX. *Rejected* because Pandoc requires installing an external system binary beyond standard Python packages (`uv`), complicating the local environment setup.
* **Alternative 2: Pure Regex replacement.** *Rejected* as highly error-prone. Nested tables and complex lists easily break regular expressions.
* **Selected Architecture: Python `markdown` library AST parsing + Custom Regex for Math.** We will use Python's built-in libraries or simple string block parsing to separate math blocks (`$$...$$`) from standard text. Standard text is sanitized, while math blocks are wrapped in `\begin{equation}`. Images are explicitly mapped to `\includegraphics[width=\textwidth]{...}`.

## 4. Test Scenarios & Acceptance Criteria
* **Scenario A (Asset Injection):** The agent outputs `Here is the cost breakdown: ![Cost Graph](assets/graph.png)`.
  * *Success:* The parser converts this specifically to `\begin{figure}[h] \centering \includegraphics[width=0.8\textwidth]{assets/graph.png} \caption{Cost Graph} \end{figure}`.
* **Scenario B (Special Character Sanitization):** The agent writes "The company grew by 50% & their revenue was $100M."
  * *Success:* The parser escapes the string to: "The company grew by 50\% \& their revenue was \$100M."
* **Scenario C (Table Formatting):** The agent outputs a standard markdown table.
  * *Success:* The parser converts it to a `\begin{tabular}` environment, ensuring column alignments (`|c|c|`) match the markdown structure.
